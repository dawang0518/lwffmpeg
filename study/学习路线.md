# FFmpeg 源码学习路线

## 一、整体架构

FFmpeg 源码分为 **核心库** 和 **命令行工具** 两大部分：

| 目录 | 作用 | 重要程度 |
|------|------|----------|
| `libavutil/` | 基础工具库（数学、内存、像素格式等） | ⭐⭐⭐ 基础 |
| `libavcodec/` | 编解码器（H.264、H.265、AAC 等） | ⭐⭐⭐⭐⭐ 核心 |
| `libavformat/` | 封装/解封装（MP4、FLV、TS 等） | ⭐⭐⭐⭐⭐ 核心 |
| `libswscale/` | 图像缩放和像素格式转换 | ⭐⭐⭐ |
| `libswresample/` | 音频重采样 | ⭐⭐⭐ |
| `libavfilter/` | 音视频滤镜 | ⭐⭐⭐ |
| `libavdevice/` | 设备输入输出 | ⭐⭐ |
| `fftools/` | ffmpeg/ffplay/ffprobe 命令行工具 | ⭐⭐⭐⭐ 入口 |

## 二、学习顺序

### 第 1 阶段：理解数据流（1-2 周）

从 `fftools/ffmpeg.c` 的 `main()` 入手，跟踪一个最简单的转码命令的执行流程：

```bash
ffmpeg -i input.mp4 output.avi
```

核心流程：**解封装 → 解码 → （滤镜处理） → 编码 → 封装**

重点文件：

| 文件 | 作用 |
|------|------|
| `fftools/ffmpeg.c` | 主入口 |
| `fftools/ffmpeg_demux.c` | 解封装 |
| `fftools/ffmpeg_dec.c` | 解码 |
| `fftools/ffmpeg_enc.c` | 编码 |
| `fftools/ffmpeg_mux.c` | 封装输出 |
| `fftools/ffmpeg_filter.c` | 滤镜处理 |
| `fftools/ffmpeg_opt.c` | 命令行选项解析 |
| `fftools/ffmpeg_sched.c` | 调度器 |

### 第 2 阶段：掌握核心数据结构（1-2 周）

这几个结构体是 FFmpeg 的骨架，必须搞懂：

| 结构体 | 头文件 | 作用 |
|--------|--------|------|
| `AVFormatContext` | `libavformat/avformat.h` | 封装/解封装上下文 |
| `AVCodecContext` | `libavcodec/avcodec.h` | 编解码器上下文 |
| `AVPacket` | `libavcodec/packet.h` | 压缩数据包 |
| `AVFrame` | `libavutil/frame.h` | 解码后的原始帧 |
| `AVStream` | `libavformat/avformat.h` | 媒体流信息 |
| `AVCodec` | `libavcodec/codec.h` | 编解码器描述 |

### 第 3 阶段：深入某个模块（按兴趣选）

- **封装格式** → 读 `libavformat/mov.c`（MP4）或 `libavformat/flvdec.c`（FLV）
- **编解码** → 读 `libavcodec/h264dec.c`（H.264 解码）或 `libavcodec/aacenc.c`（AAC 编码）
- **滤镜** → 读 `libavfilter/vf_scale.c`（缩放滤镜，比较简单，适合入门）

## 三、实践建议

### 1. 用 LLDB 调试跟踪

比纯读代码效率高很多，可以直接观察运行时的数据流：

```bash
# 使用 debug 版本（编译时生成的 _g 后缀版本）
lldb ffmpeg_g
(lldb) breakpoint set -n main
(lldb) run -i input.mp4 output.avi
```

常用断点位置：
- `avformat_open_input` — 打开输入文件
- `avformat_find_stream_info` — 探测流信息
- `avcodec_send_packet` — 发送压缩包给解码器
- `avcodec_receive_frame` — 从解码器接收原始帧
- `avcodec_send_frame` — 发送原始帧给编码器
- `avcodec_receive_packet` — 从编码器接收压缩包
- `av_interleaved_write_frame` — 写入输出文件

### 2. 写小程序调用 API

从简单到复杂，逐步深入：

1. **解封装**: 打开文件，遍历所有流，打印流信息
2. **解码**: 打开文件 → 找到视频流 → 逐帧解码 → 保存为图片
3. **转码**: 解码 → 编码为另一种格式
4. **滤镜**: 解码 → 加滤镜（如缩放、水印） → 编码

### 3. 带着问题读代码

推荐问题列表：

- "seek 是怎么实现的？" → 跟踪 `av_seek_frame()`
- "H.264 解码一帧的流程？" → 跟踪 `avcodec_send_packet()` / `avcodec_receive_frame()`
- "MP4 的 moov box 是怎么解析的？" → 读 `libavformat/mov.c`
- "音视频同步是怎么做的？" → 读 `fftools/ffplay.c`
- "硬件加速解码如何工作？" → 读 `fftools/ffmpeg_hw.c`
